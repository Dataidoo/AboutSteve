CHARTS

Same data, five ways to look at it.

Donut for the breakdown. Line for the trend. Bars for the head-to-head. Stacked for the yearly view. Change the dropdown, the chart redraws. Chart.js ships with the .exe so it runs offline.

COMPARE

Two datasets, side-by-side.

Switch to Compare and the workspace splits. Different folders, different date ranges, this-month-vs-last-month — set both sides independently, see them at once. Useful for tax season, end-of-year reviews, or confirming a gut feeling about a trend.

LOCAL FOLDERS

Point it at folders. That's the setup.

Each folder has a type — flat, yearly (year-then-month sub-folders), or one-offs — and a column mapping. Add as many folder types as you want. Everything stays on your machine.

Main → sub. Tag as deep as the data deserves.

Every row gets a main category and a sub. Add a new sub any time — even mid-way through a PDF2Sheets import — without re-categorizing what came before. No rework.

Branches go as deep as you need. Decide later that "Eating Out" should split into "Coffee" / "Lunch" / "Date Night" / "Takeout"? Add them. Old rows keep their old tag, new rows get the finer ones. Reports just work.

Built to be changed.

DalaDash doesn't care what categories you use, what columns your spreadsheets have, or how many folders you stack on top of it. Tell it once in the config; it remembers.

AI PROMPT EXAMPLE

Something you can paste straight in.

DalaDash is most of the work done. Customizing it is mostly just changing words and column letters. Ask like this:

"I've attached daladash.py and daladash.html. I want to use this to track my freelance business, not personal finance. Change the folders to: Invoices, Expenses, Mileage, Client Notes. The xlsx columns are date / client / project / category / hours / amount. Use a warm orange theme instead of the blue. Drop the 'Tax Files' section in the sidebar — I don't need it. Send me back both files updated."

The clearer you are about column letters and folder paths, the cleaner the result. The Python side rarely needs more than the DEFAULT_CONFIG block changed.
